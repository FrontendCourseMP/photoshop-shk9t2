# Photoshop-shk9t2 — GrayBit‑7 Image Lab

Live demo: https://starlit-pika-08a601.netlify.app/ (работает)

## Описание

Простой статический веб‑редактор изображений (папка `docs/`), реализующий лабораторные работы 1–5 курса.

Реализовано:

- GB7 (GrayBit‑7) — собственный кодек для тестовых изображений (encode/decode).
- Панель каналов и инструмент «Пипетка» (показывает значения цвета, включая CIELAB).
- Инструмент «Уровни» — гистограмма, создание LUT, предпросмотр перед применением.
- Изменение размера — nearest‑neighbor и билинейная интерполяция.
- Фильтры — ядро 3×3, предустановки, применение по каналам, обработка краёв (copy/black/white), реализация в Web Worker (с запасным вариантом на main thread).
- Загрузка/скачивание изображений (PNG, JPG, GB7), холст (`<canvas>`), предпросмотр с троттлингом, модальные диалоги.

## Быстрый старт

1. Клонировать репозиторий:

   git clone https://github.com/FrontendCourseMP/photoshop-shk9t2.git

2. Запустить локальный HTTP‑сервер из корня проекта (или прямо из `docs/`):

   - Через Python (рекомендуется):

     cd docs
     python -m http.server 8000

     или (Python 3.7+):

     python -m http.server 8000 --directory docs

   - Через Node (если установлен):

     npx http-server docs -p 8000

3. Открыть в браузере: http://localhost:8000/

## Файлы, представляющие интерес

- `docs/index.html` — пользовательский интерфейс.
- `docs/js/app.js` — основная логика (загрузка, отрисовка, уровни, масштаб и т.д.).
- `docs/js/gb7.js` — кодек GrayBit‑7.
- `docs/js/filters.js` — UI ↔ Worker для фильтров.
- `docs/js/filtersWorker.js` — Web Worker для выполнения свёртки 3×3.
- `.github/workflows/pages.yml` — workflow для деплоя на GitHub Pages.

## Развёртывание

- В репозитории уже есть workflow для GitHub Pages; если сайт не опубликован автоматически, откройте Settings → Pages и выберите ветку `gh-pages` (root) либо попросите администратора репозитория включить публикацию.
- Альтернатива: Netlify/Vercel — быстрый способ опубликовать `docs/` (см. live demo выше).
